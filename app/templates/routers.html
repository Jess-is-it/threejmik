How auto backups work
RouterVault checks for configuration changes every Check for changes (hours). If it detects a change, it backs up immediately. If there are no changes, it won’t create a backup every time — instead, it still makes sure you get a fresh backup at least every Backup Age Limit (days) by forcing one at the Daily backup time.

Example: Backup Age Limit = 3 days. Today is Jan 23. Daily backup time is 11:30 PM, and change checks run every 2 hours. On Jan 25 at 11:30 PM, if there are still no changes, RouterVault skips the backup. On Jan 26 at 11:30 PM, if there are still no changes, RouterVault forces a backup because it has been 3 days since the last successful backup.

You can change the Backup Age Limit in Settings.
