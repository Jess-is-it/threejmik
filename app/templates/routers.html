Auto backup guide

RouterVault checks for configuration changes every Check for changes (hours). If it detects a change, it backs up immediately.

If there are no changes, it still makes sure you get a fresh backup at least every Force backup after (days) by forcing one at the Daily backup time.

Example (real-world timing)

  • Force backup after: 3 days
  • Daily backup time: 23:30 (Asia/Manila)
  • Check for changes: every 2 hours
  • Last successful backup: Jan 23, 22:32 PH

- Jan 24/25: No changes; baselines at 23:30 see the last backup is 1–2 days old, so no forced backup.

- Jan 26: At 23:30, last backup is older than 3 days → RouterVault creates an Auto forced backup.

Any detected config change on an interval check would trigger an immediate Automatic backup and reset the clock.
